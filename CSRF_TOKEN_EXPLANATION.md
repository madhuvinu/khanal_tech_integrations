# CSRF Token Explained 🔐

## What is CSRF?

**CSRF = Cross-Site Request Forgery**

It's a security attack where a malicious website tricks your browser into making requests to a website you're logged into.

### Example Attack Scenario:
1. You're logged into `kfltest.localhost:8003` (your kiosk app)
2. You visit a malicious website `evil.com`
3. That website has code that makes your browser send a POST request to `kfltest.localhost:8003/api/method/delete_all_data`
4. Since you're logged in, your browser sends the request WITH your cookies/session
5. **Your server thinks it's you and deletes everything!** 💥

## How CSRF Token Prevents This

**CSRF Token = A secret random string that only YOUR legitimate app knows**

1. **Server generates** a unique token when you load the page
2. **Frontend includes** this token in every POST request
3. **Server verifies** the token matches before processing the request
4. **Malicious sites can't get the token** (they can't read your page's JavaScript)

## In Your Application

### 🔄 Flow Diagram:

```
1. User visits page
   ↓
2. Backend (kiosk.py) generates CSRF token
   frappe.sessions.get_csrf_token()
   ↓
3. Token injected into HTML
   window.csrf_token = "abc123xyz..."
   ↓
4. Frontend sends token with every POST request
   headers['X-Frappe-CSRF-Token'] = window.csrf_token
   ↓
5. Backend verifies token matches
   ✅ Valid → Process request
   ❌ Invalid → Return 417 error
```

## Code Flow

### Backend (Python) - GENERATES the token:

```python
# kiosk.py
def get_context(context):
    # Frappe generates a unique token for this session
    csrf_token = frappe.sessions.get_csrf_token()
    
    # Inject it into the HTML template
    context.boot.csrf_token = csrf_token
    return context
```

**What Frappe does:**
- Generates a random secret string (like `"a8f3d9c2e1b4f6..."`)
- Stores it in the session
- Returns it to the frontend

### Frontend (JavaScript) - USES the token:

```javascript
// baseService.js
ensureCSRFToken() {
  // Get token from window (injected by backend)
  const csrfToken = window.csrf_token
  return csrfToken
}

// Add to every POST request
axios.post(url, data, {
  headers: {
    'X-Frappe-CSRF-Token': csrfToken  // ← This prevents CSRF attacks!
  }
})
```

## Why We Need It

Without CSRF token:
- ❌ Any website can make requests on your behalf
- ❌ Malicious sites can delete/modify data
- ❌ Security vulnerability

With CSRF token:
- ✅ Only your legitimate app has the token
- ✅ Malicious sites can't get the token
- ✅ Server rejects requests without valid token (417 error)

## Key Points

1. **We DON'T generate it** - Frappe backend generates it
2. **We just USE it** - Frontend reads it and sends it with requests
3. **Token is unique per session** - Different for each user/login
4. **Token expires** - Gets regenerated on new page load
5. **Required for POST requests** - GET requests don't need it (usually)

## Common Issues

### 417 Error = CSRF Token Missing/Invalid
- Token not set: `window.csrf_token` is undefined
- Token expired: Page needs refresh
- Token mismatch: Server session changed

### Solution:
- Check `window.csrf_token` exists
- Refresh page to get new token
- Ensure you're logged in

## Summary

**CSRF Token = Security measure to prevent malicious websites from making requests on your behalf**

- ✅ **Generated by**: Frappe backend (`frappe.sessions.get_csrf_token()`)
- ✅ **Stored in**: `window.csrf_token` (injected by HTML template)
- ✅ **Used by**: Frontend (sent in `X-Frappe-CSRF-Token` header)
- ✅ **Verified by**: Frappe backend (automatically)
- ✅ **Purpose**: Prevent CSRF attacks

**You don't need to generate it - Frappe handles that!** You just need to make sure your frontend sends it with every POST request (which we've already done ✅).
